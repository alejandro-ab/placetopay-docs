# Códigos de razón

El siguiente apartado describe los posibles códigos de respuesta que puede retornar el sistema en las distintas operaciones de tokenización, validación y gestión de tarjetas. Estos códigos ayudan a comprender el estado de una operación y permiten identificar con claridad posibles errores o condiciones especiales.

| Código | Descripción                                                                                      | Cuándo ocurre                                                                                 |
|--------|--------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------|
| 00     | Operación exitosa. El token ha sido recuperado correctamente.                                    | Cuando la solicitud se procesa sin errores y el token se retorna satisfactoriamente.          |
| BR     | La operación no está permitida o el recurso solicitado no está disponible.                       | Cuando el token ha expirado, no existe el recurso solicitado o el comercio no tiene permisos. |
| 15     | La marca de la tarjeta no está habilitada para el comercio.                                      | Se intenta operar con una franquicia no activada.                                             |
| 31     | El banco emisor no está soportado.                                                               | Se intenta tokenizar o consultar una tarjeta de un banco no admitido.                         |
| 56     | No se encuentra el identificador del cliente.                                                    | El identificador del comercio o cliente no existe en el sistema de la marca.                  |
| 62     | La tarjeta no puede ser tokenizada.                                                              | La tarjeta no cumple con las reglas de la red para poder ser tokenizada.                      |
| 14     | El número de tarjeta ingresado no es válido o no es elegible para tokenización.                  | Fallan las validaciones de formato o elegibilidad de la tarjeta.                              |
| 55     | Fallo de autenticación.                                                                          | Las credenciales o llaves utilizadas no son válidas o están mal configuradas.                 |
| ?I     | No se encontró el token o la tarjeta consultada no existe.                                       | La red no tiene información del token o número de tarjeta enviado.                            |
| XF     | El comercio no tiene habilitada la franquicia correspondiente.                                   | Se intenta operar con una marca no habilitada para ese comercio.                              |
| SU     | No se obtuvo respuesta del emisor o hubo un error interno del proveedor.                         | Problemas de conexión o fallas técnicas en los sistemas de la red emisora.                    |
| X3     | La solicitud tiene errores de formato o contiene campos inválidos.                               | Peticiones con datos incompletos o mal formateados.                                           |
| XN     | El número de tarjeta ingresado no cumple con las validaciones esperadas.                         | Errores en el formato o estructura del número de tarjeta (PAN).                               |
| XR     | La respuesta de la red no fue reconocida o no es válida.                                         | El sistema recibió una respuesta inesperada o malformada desde la red.                        |
| RM     | El servicio recibió demasiadas peticiones en un corto tiempo (error 429).                        | Se superó el límite de solicitudes, se recomienda reintentar más tarde.                       |
| XH     | Error interno inesperado.                                                                        | Problemas generales del sistema que impiden procesar la solicitud.                            |
| ?-     | La operación se encuentra en estado pendiente.                                                   | Para operaciones asincrónicas que no finalizan inmediatamente.                                |
| 05     | Error general no controlado.                                                                     | Excepciones o fallos que no han sido categorizados específicamente.                           |
